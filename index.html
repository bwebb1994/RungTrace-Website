From symptom to source

Stop searching. Start tracing.

When the call is “Motor M300 isn’t running,” RungTrace takes you from a site-wide search to the exact blocking condition in minutes.

01

Search every controller

Enter PLC addresses, choose saved favorites, or use an optional RSLinx driver to discover and connect to multiple controllers at once. Then search controller- and program-level tags to find exactly where Motor M300 is controlled.

02

Watch it live

Add the relevant tags to one unified watch list, monitor them in real time, and safely write values back when troubleshooting calls for it.

03

Trace the cause

Select any tag on the rung. Tag Trace follows every write—even through Add-On Instructions—and reveals the bit holding the process back.

Tag Trace

Skip the cross-reference chase.

Select a tag anywhere on a rung and instantly build a visual flow of everything that writes to it. RungTrace follows the chain through routines and AOIs until the bit preventing the system from enabling is clear. Traditional cross-reference remains available when you need it.

Real-time plotting

See values change over time.

Trend tags from multiple PLCs on one chart, with selectable sampling as fast as 50 ms. Actual rates vary with network conditions and workload. Pause, inspect, compare, and export the moment the process goes wrong.

Sequence timing

Slow down the process—without slowing the machine.

Turn high-speed machine operations into a color-coded Gantt chart. Expose missed steps, sequence delays, and timing drift that happen too quickly to catch in live logic.

Hardware visibility

Spot chassis problems at a glance.

Scan backplanes across connected PLCs, identify faulted modules, inspect firmware, and compare live hardware with the L5X configuration to surface missing or mismatched equipment.
